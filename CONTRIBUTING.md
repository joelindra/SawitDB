# Contributing to SawitDB

Terima kasih sudah tertarik untuk berkontribusi di **SawitDB**! Project ini adalah milik komunitas, dari rakyat untuk rakyat.

Kami sangat menghargai setiap kontribusi, baik itu fix bug kecil, fitur baru, dokumentasi, atau sekadar memperbaiki typo.

## Code of Conduct (Tata Krama)

Singkat saja: **Jangan jadi "Oknum"**.
Bersikaplah sopan, saling menghargai, dan konstruktif. Kita di sini untuk membangun database, bukan dinasti politik.

## Cara Berkontribusi

### 1. Laporkan Bug (Lapor Pak!)
Jika menemukan masalah, silakan buat **Issue** baru. Sertakan:
- Langkah-langkah untuk mereproduksi (Steps to reproduce).
- Apa yang diharapkan vs apa yang terjadi.
- Versi Node.js dan OS Anda.

### 2. Request Fitur (Aspirasi Rakyat)
Punya ide "Infrastruktur Langit" yang masuk akal? Buat issue dengan label `enhancement`. Jelaskan kenapa fitur ini berguna bagi kemaslahatan umat.

### 3. Pull Requests (Kerja Nyata)

1. **Fork** repo ini.
2. Buat branch baru untuk fitur Anda (`git checkout -b fitur-keren-banget`).
3. Commit perubahan Anda (`git commit -m 'Menambahkan fitur anti-korupsi'`).
4. Push ke branch (`git push origin fitur-keren-banget`).
5. Buat **Pull Request**.

## Guidelines Pengembangan

- **Code Style**: Ikuti gaya penulisan yang sudah ada. Keep it simple. Jangan buat "Pasal Karet" (kode yang ambigu).
- **Testing**: Jika menambah fitur logic, tolong tambahkan test case di folder `tests/`. Kita tidak mau rilis fitur yang "asal bapak senang" tapi ternyata bug.
- **Commit Messages**: Gunakan Bahasa Inggris atau Bahasa Indonesia yang baik (sedikit slang boleh, tapi tetap jelas).
